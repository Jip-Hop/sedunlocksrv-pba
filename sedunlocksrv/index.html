Network interfaces detected by Tiny Core. Use these names with NET_IFACES and EXCLUDE_NETDEV.
Use Boot to kexec directly into the OS while keeping unlocked state. Use Reboot when hardware or firmware needs a full restart (for example, NIC firmware/switch renegotiation or when the target OS only behaves correctly after a cold reboot).
You must unlock at least one drive first in this boot session before changing passwords.
Diagnostics are pulled from sedutil-cli --query per drive.
Expert Mode: these operations can permanently destroy data or ownership metadata.
Use this only when standard unlock paths cannot recover the drive.
Revert TPer (destructive)
Attempts a TPer revert using the current drive password. This can invalidate existing unlock configuration.
PSID Revert (full erase)
Last-resort factory reset by PSID. This erases access to existing encrypted data.
