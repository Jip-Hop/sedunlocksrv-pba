Target drives
This updates Admin1 and SID on the selected drive(s). If SID password changes fail, check firmware or TPM settings and disable Block SID for the next boot. You must unlock at least one drive first in this boot session before changing passwords.
Example BIOS path: TPM or security settings with Disable Block SID set to Disabled, then accept the one-time change on the next boot.

Attempts a TPer revert using the current drive password. This can invalidate existing unlock configuration.

Last-resort factory reset by PSID. This erases access to existing encrypted data.
